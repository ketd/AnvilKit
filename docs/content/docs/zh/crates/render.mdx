`anvilkit-render` 是 AnvilKit 的核心渲染模块，基于 `wgpu 0.19` 提供统一的 2D/3D 渲染能力。

## 渲染管线

引擎每帧执行多 pass HDR 管线：

```
Pass 0  CSM 阴影 (×3 cascade)           → D2Array 深度纹理
Pass 1  HDR 场景 (PBR + MSAA 4×)        → Rgba16Float RT (resolve)
Pass 2  SSAO (半分辨率 + blur)           → R8Unorm AO 纹理
Pass 3  Bloom (降采样 ×5 + 升采样 ×4)   → Rgba16Float mip[0]
Pass 4  Tonemap (HDR × AO + Bloom → ACES Filmic → sRGB) → Swapchain
```

所有 pass 共享单个 `CommandEncoder`，最小化 CPU 开销。

## Bloom

从 HDR 值中提取亮度超过阈值的像素，产生物理正确的泛光效果。

**管线**: 13-tap 加权降采样（防止萤火虫噪点） → 5 级 mip chain → 9-tap tent filter 升采样（additive blend） → tonemap shader 中合成。

**配置** (`BloomSettings` ECS 资源):

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `enabled` | `true` | 开关 bloom（关闭时零 GPU 开销） |
| `threshold` | `1.0` | HDR 亮度阈值 |
| `knee` | `0.1` | 软过渡范围，避免硬边 |
| `intensity` | `0.3` | bloom 贡献乘数 |
| `mip_count` | `5` | 降采样级数 |

```rust
use anvilkit_render::renderer::bloom::BloomSettings;

// 运行时调节
app.insert_resource(BloomSettings {
    threshold: 0.8,
    intensity: 0.5,
    ..Default::default()
});
```

**文件**: `bloom.rs`, `bloom_downsample.wgsl`, `bloom_upsample.wgsl`

## SSAO

屏幕空间环境遮蔽 — 使用 hemisphere kernel 采样 + 深度重建法线。

**管线**: 深度 buffer → 重建法线（cross-product screen-space derivatives） → 半球采样 + 4×4 noise 旋转 → 半分辨率 AO → 4×4 box blur → tonemap 中乘入场景。

**配置** (`SsaoSettings` ECS 资源):

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `enabled` | `true` | 开关 SSAO |
| `quality` | `Medium` | 采样数: `Low`=16, `Medium`=32, `High`=64 |
| `radius` | `0.5` | view-space 采样半径 |
| `bias` | `0.025` | 深度偏移，防止平面自遮蔽 |
| `intensity` | `1.0` | AO 暗部强度乘数 |

```rust
use anvilkit_render::renderer::ssao::{SsaoSettings, SsaoQuality};

app.insert_resource(SsaoSettings {
    quality: SsaoQuality::High,
    radius: 0.3,
    ..Default::default()
});
```

**文件**: `ssao.rs`, `ssao.wgsl`, `ssao_blur.wgsl`

## 级联阴影 (CSM)

3 级 cascade shadow maps，替代单级 shadow map，距离相关的阴影质量。

**原理**: 将相机视锥体分割为 3 段（远平面的 10%、30%、100%），每段紧密包围框生成正交投影。PBR fragment shader 按 view-Z 选择最优 cascade。

| Cascade | 覆盖范围 | 质量 |
|---------|----------|------|
| 0 | 近处 (0–10% far) | 最高分辨率 |
| 1 | 中距 (10–30% far) | 中等分辨率 |
| 2 | 远处 (30–100% far) | 较低分辨率 |

**GPU 布局** (`PbrSceneUniform`):

| 字段 | 类型 | 说明 |
|------|------|------|
| `cascade_view_projs` | `[mat4×4; 3]` | 每级 cascade 的光空间矩阵 |
| `cascade_splits` | `[f32; 4]` | `[split0, split1, split2, shadow_texel_size]` |
| `emissive_factor.w` | `f32` | cascade 数量 (3.0) |

Shadow map 纹理: `Depth32Float`, `2048×2048`, `depth_or_array_layers = 3` (`D2Array`)。

**文件**: `events.rs` (`compute_cascade_matrices`), `pbr.wgsl` (`calculate_shadow`)

## PBR 材质系统

Cook-Torrance 镜面 BRDF: GGX 分布 + Smith 几何 + Fresnel-Schlick。

**Bind Groups**:

| Group | 内容 |
|-------|------|
| 0 | 场景 uniform (992 bytes): model, view_proj, normal_matrix, camera_pos, lights[8], cascade 矩阵 |
| 1 | 材质纹理: base color, normal map, metallic-roughness, AO, emissive (5 纹理 + 5 采样器) |
| 2 | IBL + 阴影: BRDF LUT, 线性采样器, CSM shadow map array, 比较采样器 |
| 3 | 关节矩阵 (skinned only): `storage<array<mat4×4, 128>>` |

## 骨骼动画

GPU 线性混合蒙皮 (Linear Blend Skinning)，最多 128 个关节。

**顶点格式** (`SkinnedVertex`, 72 字节):

| 属性 | 类型 | Location |
|------|------|----------|
| `position` | `vec3<f32>` | 0 |
| `normal` | `vec3<f32>` | 1 |
| `texcoord` | `vec2<f32>` | 2 |
| `tangent` | `vec4<f32>` | 3 |
| `joint_indices` | `u16×4` | 4 |
| `joint_weights` | `f32×4` | 5 |

**蒙皮公式** (在 `skinned_pbr.wgsl` 中):
```
skinned_pos = Σ(weight[i] × joints[index[i]] × position)  i ∈ 0..4
```

**运行时流程**:
1. `AnimationPlayer::advance(dt)` 推进 `current_time`
2. `compute_bone_matrices(skeleton, player)` 计算 `global × inverse_bind`
3. 上传 `BoneMatrices` 到 storage buffer (bind group 3)
4. 顶点着色器在 model/view-proj 变换之前应用 LBS

## UI 系统

基于 Flexbox 的 UI，GPU 加速矩形渲染，事件系统和控件构造器。

**布局**: 基于 `taffy`（完整 CSS Flexbox 子集）。`UiLayoutEngine::compute_layout()` 输入 `UiNode` 实体 + 容器尺寸，输出 `[x, y, w, h]` 计算矩形。

**渲染**: `UiRenderer` 在单次 draw call 中绘制带圆角的矩形（背景色 + 边框 + 文本），alpha 混合到 3D 场景之上。

**事件系统**:

| 事件 | 触发条件 |
|------|----------|
| `HoverEnter` | 鼠标进入节点范围 |
| `HoverLeave` | 鼠标离开节点范围 |
| `Click` | 鼠标在节点内按下 |

```rust
use anvilkit_render::renderer::ui::{UiEvents, Widget, UiNode};

// 创建控件
let button = Widget::button("开始游戏");
let panel = Widget::panel();
let label = Widget::label("得分: 100");
let row = Widget::row();     // 水平布局
let col = Widget::column();   // 垂直布局

// 检查事件
let events = world.resource::<UiEvents>();
if events.was_clicked(button_entity) {
    // 处理按钮点击
}
if events.is_hovered(button_entity) {
    // 显示悬停效果
}
```

**UiStyle 字段**: `flex_direction`, `justify_content`, `align_items`, `width`, `height`, `min_width`, `min_height`, `max_width`, `max_height`, `padding[4]`, `margin[4]`, `gap`, `flex_grow`, `flex_shrink`。

## 自定义渲染管线

不使用内置 PBR 管线的游戏（如体素游戏），可使用 `RenderApp` 低级模式配合 helper API：

```rust
use anvilkit_render::window::RenderApp;

impl ApplicationHandler for MyGame {
    fn window_event(&mut self, el, wid, ev) {
        // 游戏特有的按键处理...

        // 引擎处理输入转发（键盘、鼠标、光标、滚轮）
        RenderApp::forward_input(&mut self.app, &ev);
        self.render_app.window_event(el, wid, ev);
    }

    fn device_event(&mut self, el, did, ev) {
        // 引擎转发鼠标原始移动 → InputState.mouse_delta
        RenderApp::forward_device_input(&mut self.app, &ev);
        self.render_app.device_event(el, did, ev);
    }

    fn about_to_wait(&mut self, _el) {
        // 引擎 tick: DeltaTime → app.update() → InputState.end_frame() → request_redraw
        self.render_app.tick(&mut self.app);
        // 你的后更新逻辑...
    }
}
```

## 资源管理

`RenderAssets` 通过 Handle 系统管理 GPU 资源：

```rust
let handle = render_assets.upload_texture(device, queue, &texture_data);
let mesh_handle = render_assets.upload_mesh(device, &mesh_data);

if let Some(texture) = render_assets.get_texture(handle) {
    // 使用纹理
}
```

`BufferPool` 提供高效的 GPU buffer 复用：

```rust
let (buffer, capacity) = pool.acquire(device, needed_size, BufferUsages::VERTEX);
// ... 使用 buffer ...
pool.release(buffer, capacity, BufferUsages::VERTEX);
```
