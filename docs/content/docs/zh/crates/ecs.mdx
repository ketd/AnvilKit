`anvilkit-ecs` 基于 `bevy_ecs` 提供应用生命周期管理、Plugin 系统、System 调度、变换层次、内置物理和场景序列化。

## 应用生命周期

`App` 是应用的入口。典型的初始化流程为：

```
App::new() → add_plugins() → insert_resource() → add_systems() → run() / update()
```

```rust
use anvilkit_ecs::prelude::*;
use anvilkit_ecs::schedule::AnvilKitSchedule;

let mut app = App::new();

// 1. 注册插件（顺序重要——依赖项在前）
app.add_plugins(AnvilKitEcsPlugin);
app.add_plugins(TransformPlugin);
app.add_plugins(PhysicsPlugin);

// 2. 插入全局资源
app.insert_resource(MyConfig { difficulty: 3 });
app.init_resource::<Score>();          // 使用 Default

// 3. 注册系统到调度阶段
app.add_systems(AnvilKitSchedule::Startup, setup_scene);
app.add_systems(AnvilKitSchedule::Update, (player_input, enemy_ai));
app.add_systems(AnvilKitSchedule::PostUpdate, cleanup);

// 4a. 阻塞式主循环（内部反复调用 update()）
app.run();

// 4b. 或者手动驱动更新
for _ in 0..60 {
    app.update(); // 一帧
}
```

### App API

| 方法 | 签名 | 说明 |
|------|------|------|
| `new` | `() -> Self` | 创建 App，预注册所有引擎调度 |
| `add_plugins` | `<P: Plugin>(P) -> &mut Self` | 注册插件（唯一插件自动去重） |
| `add_systems` | `(impl ScheduleLabel, impl IntoSystemConfigs) -> &mut Self` | 将系统添加到调度 |
| `insert_resource` | `<R: Resource>(R) -> &mut Self` | 插入全局资源 |
| `init_resource` | `<R: Resource + FromWorld>() -> &mut Self` | 用 Default/FromWorld 初始化资源 |
| `run` | `(&mut self)` | 阻塞式主循环，直到调用 `exit()` |
| `update` | `(&mut self)` | 执行一帧：首次调用运行 Startup，之后每帧 PreUpdate -> Update -> PostUpdate |
| `exit` | `(&mut self)` | 标记在当前帧结束后退出 |

## Plugin

Plugin 封装可复用的引擎功能。实现 `Plugin` trait：

```rust
pub struct PhysicsPlugin;

impl Plugin for PhysicsPlugin {
    fn build(&self, app: &mut App) {
        app.init_resource::<DeltaTime>()
           .init_resource::<CollisionEvents>()
           .add_systems(AnvilKitSchedule::Update, (
               velocity_integration_system,
               collision_detection_system.after(velocity_integration_system),
           ));
    }
}
```

唯一插件（默认行为）最多注册一次；重复调用 `add_plugins` 会被静默跳过。

## 调度阶段

`App::update()` 每帧按以下顺序运行调度：

| 调度 | 变体 | 运行时机 | 典型用途 |
|------|------|----------|----------|
| **Startup** | `AnvilKitSchedule::Startup` | 仅首次 `update()` | 场景初始化、资源加载 |
| **PreUpdate** | `AnvilKitSchedule::PreUpdate` | 每帧 | 输入轮询、时间更新 |
| **Update** | `AnvilKitSchedule::Update` | 每帧 | 游戏逻辑、AI、物理 |
| **PostUpdate** | `AnvilKitSchedule::PostUpdate` | 每帧 | 变换传播、渲染准备 |
| **Cleanup** | `AnvilKitSchedule::Cleanup` | 每帧 | 清理临时数据、诊断 |

## SystemSet

在调度内部，系统可通过 `AnvilKitSystemSet` 分组排序：

| SystemSet | 用途 |
|-----------|------|
| `Input` | 键盘、鼠标、手柄处理 |
| `Time` | 时间/计时器更新、帧率追踪 |
| `Physics` | 速度积分、碰撞检测、物理步进 |
| `GameLogic` | AI、状态机、游戏规则 |
| `Transform` | 层次传播、坐标同步 |
| `Render` | 可见性剔除、材质更新、绘制准备 |
| `Audio` | 音频播放、音量控制 |
| `UI` | 布局、事件处理、控件更新 |
| `Network` | 同步复制、序列化、连接管理 |
| `Debug` | Gizmo、性能叠加层、日志 |

```rust
app.add_systems(AnvilKitSchedule::Update, (
    apply_gravity.in_set(AnvilKitSystemSet::Physics),
    check_collisions.in_set(AnvilKitSystemSet::Physics)
        .after(apply_gravity),
    handle_input.in_set(AnvilKitSystemSet::Input),
));
```

## System

系统是纯函数，通过参数声明数据访问需求：

```rust
fn move_system(
    time: Res<Time>,
    mut query: Query<(&Velocity, &mut Transform)>,
) {
    let dt = time.delta_seconds();
    for (vel, mut transform) in &mut query {
        transform.translation += vel.linear * dt;
    }
}

fn collision_handler(events: Res<CollisionEvents>) {
    for event in events.iter() {
        println!("碰撞: {:?} <-> {:?}", event.a, event.b);
    }
}
```

## Bundle

Bundle 将组件分组以便批量创建实体：

```rust
#[derive(Bundle)]
struct PlayerBundle {
    transform: Transform,
    global_transform: GlobalTransform,
    velocity: Velocity,
    collider: AabbCollider,
    player: Player,
```

## 变换层次

`TransformPlugin` 在 **PostUpdate** 的 `AnvilKitSystemSet::Transform` 中注册 `sync_simple_transforms` 和 `propagate_transforms`。

### 核心组件

| 组件 | 说明 |
|------|------|
| `Transform` | 相对于父实体的本地变换（translation, rotation, scale） |
| `GlobalTransform` | 世界空间变换矩阵，由传播系统计算 |
| `Parent(Entity)` | 父实体引用 |
| `Children` | 子实体列表 |

### 生成父子层次结构

```rust
use anvilkit_ecs::prelude::*;
use glam::Vec3;

fn setup_hierarchy(mut commands: Commands) {
    // 在世界坐标 (10, 0, 0) 生成父实体
    let parent = commands.spawn((
        Transform::from_xyz(10.0, 0.0, 0.0),
        GlobalTransform::default(),
        Children::empty(),
    )).id();

    // 生成相对父实体偏移 (5, 0, 0) 的子实体
    let child = commands.spawn((
        Transform::from_xyz(5.0, 0.0, 0.0),
        GlobalTransform::default(),
        Parent::new(parent),
    )).id();

    // 传播后，子实体的 GlobalTransform.translation() == (15, 0, 0)
}
```

### TransformHierarchy 工具方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `set_parent` | `(Commands, child: Entity, parent: Entity)` | 建立父子关系 |
| `remove_parent` | `(Commands, child: Entity)` | 解除子实体与父实体的关系 |
| `get_ancestors` | `(&World, Entity) -> Vec<Entity>` | 向上遍历层次到根 |
| `get_descendants` | `(&World, Entity) -> Vec<Entity>` | 递归收集所有后代 |

## 物理

### 内置 AABB 物理（默认）

`PhysicsPlugin` 在 `Update` 中注册速度积分和 N 平方 AABB 碰撞检测。

| 类型 | 种类 | 说明 |
|------|------|------|
| `DeltaTime` | 资源 | 帧间隔秒数（默认 1/60） |
| `Velocity` | 组件 | `linear: Vec3`、`angular: Vec3` |
| `AabbCollider` | 组件 | `half_extents: Vec3` |
| `CollisionEvents` | 资源 | 每帧填充的 `Vec<CollisionEvent>` |
| `CollisionEvent` | 结构体 | `a: Entity`、`b: Entity` |

| 系统 | 说明 |
|------|------|
| `velocity_integration_system` | `transform.translation += velocity.linear * dt` |
| `collision_detection_system` | N 平方 AABB 重叠检测，写入 `CollisionEvents` |

```rust
use anvilkit_ecs::prelude::*;
use anvilkit_ecs::physics::*;

fn setup_physics(mut commands: Commands) {
    // 动态运动盒子
    commands.spawn((
        Transform::from_xyz(0.0, 10.0, 0.0),
        Velocity::linear(Vec3::new(0.0, -5.0, 0.0)),
        AabbCollider::cube(0.5),
    ));

    // 静态地面
    commands.spawn((
        Transform::from_xyz(0.0, 0.0, 0.0),
        AabbCollider::new(Vec3::new(50.0, 0.1, 50.0)),
    ));
}

fn read_collisions(events: Res<CollisionEvents>) {
    for event in events.iter() {
        log::info!("碰撞: {:?} <-> {:?}", event.a, event.b);
    }
}
```

### Rapier 集成（feature `"rapier"`）

启用 `rapier` feature 以获取基于 `rapier3d` 的完整刚体模拟。

`RapierPhysicsPlugin` 在 `Update` 中注册四个系统：

| 系统 | 执行顺序 | 说明 |
|------|----------|------|
| `sync_to_rapier_system` | 第 1 | 从 ECS `RigidBody`+`Collider`+`Transform` 创建 rapier 刚体 |
| `step_physics_system` | 第 2 | 执行 rapier 模拟步进 |
| `sync_from_rapier_system` | 第 3 | 将 rapier 位置写回 ECS `Transform` |
| `extract_collision_events_system` | 第 3 | 从 rapier 窄阶段填充 `CollisionEvents` |

| 类型 | 说明 |
|------|------|
| `RapierContext` | 资源：持有所有 rapier 状态（刚体集、碰撞体集、管线等） |
| `RigidBody` | 组件：`body_type`、`mass`、`gravity_scale`、`is_sensor`、阻尼 |
| `Collider` | 组件：`shape`（Sphere/Cuboid/Capsule/TriMesh）、`friction`、`restitution` |
| `RaycastHit` | 结果结构体：`entity`、`point`、`normal`、`distance` |

```rust
use anvilkit_ecs::prelude::*;
use anvilkit_ecs::physics::*;

fn setup_rapier(mut commands: Commands) {
    // 动态球体
    commands.spawn((
        Transform::from_xyz(0.0, 10.0, 0.0),
        RigidBody::dynamic(),
        Collider::sphere(0.5),
        Velocity::default(),
    ));

    // 静态地面
    commands.spawn((
        Transform::default(),
        RigidBody::fixed(),
        Collider::cuboid(Vec3::new(50.0, 0.1, 50.0)),
    ));
}

// 射线检测查询
fn shoot_ray(ctx: Res<RapierContext>) {
    if let Some(hit) = ctx.raycast(
        Vec3::new(0.0, 5.0, 0.0), // 起点
        Vec3::NEG_Y,               // 方向
        100.0,                     // 最大距离
    ) {
        log::info!(
            "命中实体 {:?}，位置 {}（距离 {}）",
            hit.entity, hit.point, hit.distance
        );
    }
}
```

## 场景序列化（feature `"serde"`）

`SceneSerializer` 将标记了 `Serializable` 组件的实体保存/加载为 RON 文件。

| 类型 | 说明 |
|------|------|
| `Serializable` | 标记组件——只有带此组件的实体才会被序列化 |
| `SceneSerializer::save` | `(&mut World, path: &str) -> Result<usize, String>` — 返回实体数 |
| `SceneSerializer::load` | `(&mut World, path: &str) -> Result<usize, String>` — 生成实体，返回数量 |

```rust
use anvilkit_ecs::prelude::*;
use anvilkit_ecs::scene::{SceneSerializer, Serializable};

// 标记实体以便序列化
fn setup(mut commands: Commands) {
    commands.spawn((
        Transform::from_xyz(1.0, 2.0, 3.0),
        GlobalTransform::default(),
        Serializable,
    ));
}

// 保存场景
fn save_game(world: &mut World) {
    let count = SceneSerializer::save(world, "scene.ron").unwrap();
    log::info!("已保存 {} 个实体", count);
}

// 加载到新世界
fn load_game(world: &mut World) {
    let count = SceneSerializer::load(world, "scene.ron").unwrap();
    log::info!("已加载 {} 个实体", count);
}
```

RON 文件存储 `SerializedScene { version: u32, entities: Vec<SerializedEntity> }`，每个实体的 `Transform` 以 `TransformData { translation: [f32; 3], rotation: [f32; 4], scale: [f32; 3] }` 形式保存。
