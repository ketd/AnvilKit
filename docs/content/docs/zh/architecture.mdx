## 分层架构

AnvilKit 采用分层模块化设计，每层只依赖下层：

```
┌──────────────────────────────────────────────┐
│            游戏层 (games/)                    │
│        billiards / craft / your_game         │
├──────────────────────────────────────────────┤
│            系统层（高级）                     │
│    anvilkit-camera   anvilkit-audio          │
│    anvilkit-input    anvilkit-assets         │
├──────────────────────────────────────────────┤
│            核心层（低级）                     │
│    anvilkit-core     anvilkit-ecs            │
│    anvilkit-render                           │
├──────────────────────────────────────────────┤
│            第三方依赖                        │
│    bevy_ecs / wgpu / rapier / glam / taffy   │
└──────────────────────────────────────────────┘
```

## Crate 依赖图

```
anvilkit-core (数学, 时间, 错误, 持久化)
    ↑
anvilkit-ecs (App, Plugin, Schedule, Transform, Physics)
    ↑           ↑
anvilkit-render  anvilkit-input (InputState, ActionMap)
    ↑               ↑
anvilkit-camera (控制器, 特效)

anvilkit-assets (glTF, 动画, AssetServer) → anvilkit-core
anvilkit-audio (AudioSource, 空间音频) → anvilkit-ecs
```

## 每帧生命周期

```
┌─ 事件循环 (winit) ────────────────────────────────────────────┐
│                                                                │
│  1. window_event / device_event                                │
│     ├─ forward_input()     → InputState (键盘, 鼠标, 滚轮)    │
│     └─ forward_device_input() → InputState.mouse_delta         │
│                                                                │
│  2. about_to_wait                                              │
│     ├─ tick() {                                                │
│     │    DeltaTime = 实际经过时间                               │
│     │    app.update() {                                        │
│     │       Main:       (预留)                                 │
│     │       PreUpdate:  DeltaTime 同步, 父子层级同步           │
│     │       Update:     输入 → 物理 → 相机特效 → 相机控制     │
│     │       PostUpdate: Transform 传播 → 渲染提取              │
│     │       Cleanup:    (预留)                                 │
│     │    }                                                     │
│     │    InputState.end_frame()                                │
│     │    window.request_redraw()                               │
│     │ }                                                        │
│     └─ post_update (游戏特有: chunk, raycast 等)               │
│                                                                │
│  3. RedrawRequested                                            │
│     └─ render_frame() {                                        │
│          Shadow Pass ×3 (CSM cascade)                          │
│          HDR 场景 Pass (PBR + MSAA 4×)                         │
│          SSAO Pass (半分辨率 + blur)                           │
│          Bloom Pass (降采样 ×5 + 升采样 ×4)                    │
│          Tonemap Pass (HDR × AO + bloom → ACES → sRGB)        │
│          HUD Pass (线段, 文本, UI)                             │
│        }                                                       │
└────────────────────────────────────────────────────────────────┘
```

## ECS Schedule 阶段

| 阶段 | 时机 | 典型系统 |
|------|------|----------|
| `Startup` | 初始化时运行一次 | 资源设置，初始生成 |
| `Main` | 每帧最先执行 | (预留，用于全局逻辑) |
| `PreUpdate` | 每帧 Update 之前 | DeltaTime 同步、父子层级同步 |
| `Update` | 每帧主逻辑 | 输入、物理、游戏逻辑、相机 |
| `PostUpdate` | 每帧 Update 之后 | Transform 传播、渲染提取 |
| `Cleanup` | 每帧最后 | (预留) |

## Feature Flags

| Crate | Feature | 启用功能 |
|-------|---------|----------|
| `anvilkit-core` | `serde` | Transform, GlobalTransform 的序列化 |
| `anvilkit-core` | `persistence` | SaveManager, Settings, WorldStorage |
| `anvilkit-core` | `bevy_ecs` | 数学类型的 Component derive |
| `anvilkit-ecs` | `serde` | Parent, Children 序列化，场景序列化 |
| `anvilkit-ecs` | `rapier` | rapier3d 物理引擎运行时集成 |
| `anvilkit-assets` | `hot-reload` | notify 文件监视热重载 |
| `anvilkit-assets` | `bevy_ecs` | Skeleton, AnimationPlayer 的 Component derive |

## 模块概览

### anvilkit-core
基础类型库: 数学工具 (Vec3, Mat4, Transform, GlobalTransform, 插值), 时间系统, 错误类型 (AnvilKitError，`with_context` 保留原始错误类别), 持久化系统 (SaveManager, Settings, WorldStorage — 均返回 `Result<_, AnvilKitError>`)。

### anvilkit-ecs
基于 `bevy_ecs 0.14` 的应用框架: App 生命周期, Plugin 系统（含唯一性检查）, Schedule 调度 (Main/Startup/PreUpdate/Update/PostUpdate/Cleanup), AppExit 优雅退出, Transform 层级传播（自动同步）, 物理（缩放感知 AABB + rapier3d）, 场景序列化 (RON)。

### anvilkit-render
统一 2D/3D 渲染管线: PBR + HDR + Bloom + SSAO + CSM + IBL + MSAA + 骨骼动画 + 视锥剔除 + 实例化 + Sprite/Line/Particle/Text/UI。

### anvilkit-assets
资产管线: glTF 加载, AssetServer (同步+异步), 热重载, 过程化网格生成。

### anvilkit-audio
基于 `rodio` 的音频: BGM/SFX, 空间音频, 音量控制。

### anvilkit-input
输入抽象: InputState, ActionMap, RenderApp 辅助方法。

### anvilkit-camera
相机系统: 第一/第三人称/自由控制器, CameraEffects, 万向锁保护。

## 坐标系

AnvilKit 使用**右手坐标系**（与 wgpu/glam 默认一致）：+X 向右, +Y 向上, -Z 指向屏幕内。
