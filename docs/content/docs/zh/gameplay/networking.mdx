网络模块提供进程内的多人游戏网络抽象：配置、事件分发、可靠/不可靠通道、带增量压缩的 ECS 状态复制，以及客户端预测与回滚。

## 配置

### NetworkConfig（Resource）

| 字段 | 类型 | 默认值 | 说明 |
|-------|------|---------|-------------|
| `role` | `NetworkRole` | `Server` | `Server` 或 `Client` |
| `bind_addr` | `String` | `"0.0.0.0:7777"` | 绑定地址 |
| `server_addr` | `Option<String>` | `None` | 服务器地址（仅客户端） |
| `tick_rate` | `u32` | `20` | 服务器每秒 tick 数 |
| `max_clients` | `u32` | `32` | 最大连接客户端数 |

### NetworkState（Resource）

| 字段 | 类型 | 说明 |
|-------|------|-------------|
| `connected` | `bool` | 连接是否已建立 |
| `client_id` | `Option<u64>` | 分配的客户端 ID |
| `rtt_ms` | `f32` | 往返时间（毫秒） |
| `bytes_sent` / `bytes_received` | `u64` | 流量计数器 |

## 事件

`NetworkEvents` 资源每帧收集事件（由 `network_events_cleanup_system` 自动清除）：

| 事件 | 字段 |
|-------|--------|
| `Connected` | `client_id: u64` |
| `Disconnected` | `client_id: u64` |
| `DataReceived` | `client_id: u64, channel: u8, data: Vec<u8>` |

```rust
use anvilkit_ecs::network::{NetworkEvents, NetworkEvent};

fn handle_events(events: Res<NetworkEvents>) {
    for event in events.iter() {
        match event {
            NetworkEvent::Connected { client_id } => { /* ... */ }
            NetworkEvent::Disconnected { client_id } => { /* ... */ }
            NetworkEvent::DataReceived { client_id, channel, data } => { /* ... */ }
        }
    }
}
```

## 传输通道

### ReliableChannel

使用序列号和 ACK 位域追踪已发送数据包。支持基于超时的重传。

| 方法 | 说明 |
|--------|-------------|
| `send(data, current_time) -> PacketHeader` | 分配序列号，加入 ACK 追踪队列 |
| `receive_ack(ack, ack_bitfield)` | 移除已确认的数据包 |
| `get_resend_packets(current_time, timeout)` | 获取需要重传的数据包 |

### UnreliableChannel

分配序列号但不进行 ACK 追踪——即发即弃的投递方式。

## ECS 复制

| 类型 | 说明 |
|------|-------------|
| `Replicated` | 标记组件，用于标识需要复制的实体 |
| `NetworkId` | 带唯一 `id: u64` 的组件 |
| `ReplicationBuffer` | 收集 `(network_id, delta_bytes)` 变更的 Resource |

### 增量压缩

`DeltaEncoder` 提供简单的基于 XOR 的增量压缩：

```rust
use anvilkit_ecs::network::DeltaEncoder;

let old_state = vec![10, 20, 30, 40];
let new_state = vec![10, 25, 30, 50];
let delta = DeltaEncoder::encode(&old_state, &new_state);
let restored = DeltaEncoder::decode(&old_state, &delta);
assert_eq!(restored, new_state);
```

## 客户端预测

### PredictionState\<T\>

用于客户端预测和服务器校正的泛型状态历史缓冲区：

| 方法 | 说明 |
|--------|-------------|
| `push(tick, state)` | 存储一个快照 |
| `get(tick) -> Option<&T>` | 获取某个 tick 的状态 |
| `rollback_to(tick)` | 移除给定 tick 之后的所有状态 |

### InputBuffer（Resource）

存储每个 tick 的序列化输入，用于校正期间的重放：

| 字段 | 类型 | 说明 |
|-------|------|-------------|
| `inputs` | `VecDeque<(u32, Vec<u8>)>` | (tick, 序列化输入) |
| `current_tick` | `u32` | 当前模拟 tick |

## NetworkPlugin

在 `PreUpdate` 中注册所有网络资源和 `network_events_cleanup_system`：

```rust
use anvilkit_ecs::network::NetworkPlugin;

app.add_plugins(NetworkPlugin);
```
