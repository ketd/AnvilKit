## 变换层级

`TransformPlugin` 在 **PostUpdate** 的 `AnvilKitSystemSet::Transform` 下注册 `sync_simple_transforms` 和 `propagate_transforms`。此外还在 **PreUpdate** 中注册 `parent_child_sync_system`，传播每帧运行。

### 核心组件

| 组件 | 说明 |
|-----------|-------------|
| `Transform` | 相对于父级的局部变换（平移、旋转、缩放） |
| `GlobalTransform` | 世界空间变换矩阵，由传播系统计算 |
| `Parent(Entity)` | 对父实体的引用 |
| `Children` | 子实体列表 |

### 生成父子层级

```rust
use anvilkit_ecs::prelude::*;
use glam::Vec3;

fn setup_hierarchy(mut commands: Commands) {
    // 在世界位置 (10, 0, 0) 生成父实体
    let parent = commands.spawn((
        Transform::from_xyz(10.0, 0.0, 0.0),
        GlobalTransform::default(),
        Children::empty(),
    )).id();

    // 生成相对父级偏移 (5, 0, 0) 的子实体
    let child = commands.spawn((
        Transform::from_xyz(5.0, 0.0, 0.0),
        GlobalTransform::default(),
        Parent::new(parent),
    )).id();

    // 传播后，child 的 GlobalTransform.translation() == (15, 0, 0)
}
```

### TransformHierarchy 工具方法

| 方法 | 签名 | 说明 |
|--------|-----------|-------------|
| `set_parent` | `(Commands, child: Entity, parent: Entity)` | 建立父子关系（`Children` 由 `parent_child_sync_system` 自动同步） |
| `remove_parent` | `(Commands, child: Entity)` | 将子实体从父级分离 |
| `get_ancestors` | `(&World, Entity) -> Vec<Entity>` | 沿层级向上遍历到根节点（环检测，cycle-safe） |
| `get_descendants` | `(&World, Entity) -> Vec<Entity>` | 递归收集所有后代（深度限制 128） |

## 场景序列化（feature `"serde"`）

`SceneSerializer` 将标记了 `Serializable` 组件的实体保存/加载到 RON 文件。

| 类型 | 说明 |
|------|-------------|
| `Serializable` | 标记组件——只有带此组件的实体会被序列化 |
| `SceneSerializer::save` | `(&mut World, path: &str) -> Result<usize, AnvilKitError>` -- 返回实体数量 |
| `SceneSerializer::load` | `(&mut World, path: &str) -> Result<usize, AnvilKitError>` -- 生成实体，返回数量 |

```rust
use anvilkit_ecs::prelude::*;
use anvilkit_ecs::scene::{SceneSerializer, Serializable};

// 标记需要序列化的实体
fn setup(mut commands: Commands) {
    commands.spawn((
        Transform::from_xyz(1.0, 2.0, 3.0),
        GlobalTransform::default(),
        Serializable,
    ));
}

// 保存场景
fn save_game(world: &mut World) {
    let count = SceneSerializer::save(world, "scene.ron").unwrap();
    log::info!("Saved {} entities", count);
}

// 加载到新的 World
fn load_game(world: &mut World) {
    let count = SceneSerializer::load(world, "scene.ron").unwrap();
    log::info!("Loaded {} entities", count);
}
```

RON 文件存储 `SerializedScene { version: u32, entities: Vec<SerializedEntity> }`，其中每个实体以 `TransformData { translation: [f32; 3], rotation: [f32; 4], scale: [f32; 3] }` 的形式捕获其 `Transform`。

## 序列化注册表

注册自定义组件类型用于场景序列化：

```rust
use anvilkit_ecs::scene::SerializableRegistry;

let mut app = App::new();
app.register_serializable::<Health>("Health");
app.register_serializable::<Inventory>("Inventory");

// 或直接操作注册表：
let mut registry = SerializableRegistry::new();
registry.register::<Health>("Health");
assert!(registry.is_registered("Health"));
```

### 层级保留

场景序列化保留父子关系：

- `parent_index`：引用 `entities` 数组中父实体的位置
- `name` / `tag`：从 `Name` 和 `Tag` 组件捕获
- `custom_data`：`HashMap<String, String>` 供用户扩展

序列化器使用两遍方法：第一遍创建所有实体，第二遍恢复 `Parent`/`Children` 组件。
