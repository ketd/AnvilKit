Craft is a Minecraft-style voxel game demonstrating custom rendering pipelines, chunk management, and world generation.

## Project Structure

```
games/craft/
├── src/
│   ├── main.rs         # Entry + main loop
│   ├── lib.rs          # Module exports
│   ├── block.rs        # Block type definitions
│   ├── chunk.rs        # Chunk data structure
│   ├── components.rs   # ECS components
│   ├── mesh.rs         # Chunk mesh generation
│   ├── raycast.rs      # Ray picking
│   ├── resources.rs    # Game resources
│   ├── vertex.rs       # Custom vertex format
│   ├── world_gen.rs    # Procedural terrain generation
│   ├── persistence.rs  # World save/load
│   ├── render/         # Custom wgpu pipeline
│   └── systems/        # Game systems
├── assets/
│   ├── voxel.wgsl      # Voxel shader
│   └── craft_tonemap.wgsl
```

## Core Features

### Procedural World Generation

Uses Perlin noise to generate heightmaps with layered block filling:

- Surface: grass blocks
- Underground: stone, dirt
- Deep: bedrock
- Decorations: trees, flowers

### Chunk System

- Chunk size: 32 x 256 x 32
- Dynamic loading/unloading based on player position
- Greedy mesh merging to reduce triangle count
- Frustum culling (bounding sphere radius ~130)

### Custom Render Pipeline

Bypasses the PBR pipeline with a simple 2-bind-group setup:
- Group 0: scene uniform (view-projection, camera position, light direction, fog)
- Group 1: texture atlas + sampler

### Block Operations

- **Place block** — right click
- **Break block** — left click
- **Ray picking** — cast ray from camera, traverse using DDA algorithm

## v0.3 Features

Craft was updated in v0.3 to showcase the new engine systems:

### Health System

The player has a `Health` component (20 HP with passive regen). Environmental hazards apply damage through the ECS event system:

- **Fall damage** — landing velocity converted to `DamageEvent`
- **Drowning** — submerged blocks tick damage over time
- **Health regen** — slow passive healing when not taking damage
- **Death & respawn** — `DeathEvent` triggers position reset and full heal

A color-coded health bar (green/yellow/red) is rendered above the hotbar.

### Inventory

A 9-slot `SlotInventory` with stackable items replaces the old fixed block palette. Breaking a block adds it to inventory; placing consumes one. The hotbar HUD reflects actual inventory contents.

### ActionMap

All key bindings are registered via `ActionMap` (logical action to physical binding). This enables future support for user-configurable controls via `ActionMap::apply_overrides()`.

### Data-Driven Blocks

Block types are defined in `assets/data/blocks.ron` using `DataTable`. The `BlockDefCache` is built from the table at startup, decoupling block properties from code.

### Locale / i18n

HUD text and block names are looked up through `Locale` (`assets/data/locale_en.ron`). The system supports fallback chains, making it straightforward to add translations.

### Player Persistence

Player state (position, health, inventory, flying mode, selected slot, day/night time) is saved and restored via `Ctrl+S`. World modifications (placed/broken blocks) are also persisted.
