## Key APIs

### Canvas2D

Immediate-mode 2D drawing API in `anvilkit_render::renderer::canvas2d`.

| Type | Description |
|------|-------------|
| `Canvas2DRenderer` | GPU resources for 2D rendering. Created once in `init()`. |
| `Canvas2D` | Per-frame drawing context obtained via `Canvas2D::begin()`. |

Methods on `Canvas2D`: `clear()`, `draw_rect()`, `draw_text()`, `draw_sprite()`, `finish()`.

### Canvas3D

Scene-graph 3D API in `anvilkit_render::renderer::canvas3d`.

| Type | Description |
|------|-------------|
| `Canvas3DRenderer` | GPU resources for 3D rendering. Created once in `init()`. |
| `Canvas3D` | Per-frame 3D scene context obtained via `Canvas3D::begin()`. |

Methods on `Canvas3D`: `clear()`, `spawn_mesh()`, `set_camera()`, `add_light()`, `finish()`.

### Frame Capture

Both Canvas2D and Canvas3D support `capture_frame()`, which returns the rendered frame as raw image bytes. This is the foundation of the AI-first design: agents can see what the player sees.

### MCP Tools (anvilkit-mcp)

The MCP server exposes 17 tools organized into four categories:

| Category | Tools |
|----------|-------|
| **Visual** | `capture_frame`, `describe_scene`, `get_screen_info` |
| **Control** | `press_key`, `release_key`, `move_mouse`, `click`, `scroll` |
| **State** | `get_component`, `set_component`, `list_entities`, `get_resource` |
| **Scene** | `spawn_object`, `despawn_object`, `save_scene`, `load_scene`, `run_system` |

## Building Docs Locally

To generate and view the API documentation locally:

```bash
cargo doc --workspace --no-deps --open
```

This builds documentation for all workspace crates and opens it in your browser.
