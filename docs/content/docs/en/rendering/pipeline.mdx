`anvilkit-render` is AnvilKit's core rendering module, providing unified 2D/3D rendering capabilities based on `wgpu 0.19`.

## Entry Points

AnvilKit offers two agent-friendly rendering APIs that sit above the full PBR pipeline:

- **Canvas2D** — Immediate-mode 2D drawing (rects, text, sprites). Best for prototypes, UI, and 2D games. AI agents can draw directly via MCP tools.
- **Canvas3D** — Scene-graph 3D API with PBR materials, lights, and camera control. Agents can spawn objects, set materials, and capture frames without touching the low-level pipeline.
- **Frame Capture** — Both Canvas2D and Canvas3D support `capture_frame()`, returning the rendered image as bytes. This is what enables the AI-first promise: the agent can see what the player sees.

For full control, use the multi-pass PBR pipeline described below.

## Render Pipeline

The engine runs a multi-pass HDR pipeline per frame:

All passes share a single `CommandEncoder` per frame for minimal CPU overhead.

### v0.3 Render Restructuring

In v0.3, large monolithic files were split into organized directories:

- **`events.rs`** (1414 lines) split into `events/` directory: `mod.rs`, `lighting.rs`, `render_app.rs`, `gpu_init.rs`, `render_loop.rs`, `input.rs`
- **`draw.rs`** (568 lines) split into `draw/` directory: `mod.rs`, `culling.rs`, `lighting.rs`, `commands.rs`, `gpu.rs`

### Shared Utilities (v0.3)

- **`CachedBuffer`**: A shared GPU buffer caching utility that replaces 5 duplicate `cached_vb` patterns previously scattered across the ui/text/sprite/line/particle renderers.
- **`ProjectionUniform`**: A shared type that replaces 5 duplicate 64-byte ortho/scene uniform structs.
- **`DebugRenderer`** now includes `LineRenderer` functionality (the two were merged in v0.3; see [Debug Renderer](/docs/en/devtools/debug-renderer)).

## Resource Management

`RenderAssets` manages GPU resources through a Handle system:

```rust
let handle = render_assets.upload_texture(device, queue, &texture_data);
let mesh_handle = render_assets.upload_mesh(device, &mesh_data);

if let Some(texture) = render_assets.get_texture(handle) {
    // Use texture
}
```

`BufferPool` provides efficient GPU buffer reuse:

```rust
let (buffer, capacity) = pool.acquire(device, needed_size, BufferUsages::VERTEX);
// ... use buffer ...
pool.release(buffer, capacity, BufferUsages::VERTEX);
```

## PostProcessSettings

Unified post-processing configuration as an ECS resource:

```rust
use anvilkit_render::renderer::post_process::PostProcessSettings;

app.insert_resource(PostProcessSettings {
    bloom: Some(BloomSettings { intensity: 0.3, threshold: 1.0 }),
    ssao: Some(SsaoSettings { radius: 0.5, bias: 0.025 }),
    dof: None,           // disabled
    motion_blur: None,   // disabled
    color_grading: None, // disabled
    ao_input_enabled: true,
    ..Default::default()
});
```

Execution order: **SSAO → DOF → Motion Blur → Bloom → Color Grading → Tonemap**

Each stage is skipped when its `Option` is `None`.

## StandardMaterial

High-level PBR material component with builder pattern:

```rust
use anvilkit_render::renderer::standard_material::StandardMaterial;

commands.spawn((
    mesh_handle,
    StandardMaterial::new()
        .with_base_color([0.8, 0.2, 0.1, 1.0])
        .with_metallic(0.9)
        .with_roughness(0.1),
    Transform::default(),
    GlobalTransform::default(),
));
```

`render_extract_system` queries `(MeshHandle, StandardMaterial, GlobalTransform)` and auto-generates `DrawCommand`s. Entities with `StandardMaterial` use the default PBR pipeline via `DefaultMaterialHandle`.

## PipelineCache / BindGroupCache

GPU resource deduplication:

- **PipelineCache**: Caches render pipelines by `PipelineKey` (vertex_format, BlendMode, CullMode). Uses `get_or_create()` to avoid redundant GPU pipeline creation.
- **BindGroupCache**: Caches material bind groups by material ID with a dirty flag for invalidation. Dirty entries are automatically recreated on next access.

```rust
let mut cache = PipelineCache::new();
let handle = cache.get_or_create(key, |k| create_pipeline(device, k));
```

## Camera Projection

`CameraComponent` supports perspective and orthographic projection:

```rust
use anvilkit_render::plugin::{CameraComponent, Projection};

// Perspective (default)
let camera = CameraComponent::default();

// Orthographic (2D, UI)
let ortho_camera = CameraComponent {
    projection: Projection::Orthographic {
        left: -10.0, right: 10.0, bottom: -10.0, top: 10.0,
    },
    ..Default::default()
};

// Multi-camera: higher priority renders first
let minimap = CameraComponent { priority: 10, ..Default::default() };
```
