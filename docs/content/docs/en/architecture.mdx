## Layered Architecture

AnvilKit uses a layered modular design where each layer depends only on layers below it:

```
┌──────────────────────────────────────────────┐
│            Game Layer (games/)                │
│        billiards / craft / your_game         │
├──────────────────────────────────────────────┤
│          System Layer (High-level)           │
│    anvilkit-camera   anvilkit-audio          │
│    anvilkit-input    anvilkit-assets         │
├──────────────────────────────────────────────┤
│           Core Layer (Low-level)             │
│    anvilkit-core     anvilkit-ecs            │
│    anvilkit-render                           │
├──────────────────────────────────────────────┤
│          Third-party Dependencies            │
│    bevy_ecs / wgpu / rapier / glam / taffy   │
└──────────────────────────────────────────────┘
```

## Crate Dependency Graph

```
anvilkit-core (math, time, error, persistence)
    ↑
anvilkit-ecs (App, Plugin, Schedule, Transform, Physics)
    ↑           ↑
anvilkit-render  anvilkit-input (InputState, ActionMap)
    ↑               ↑
anvilkit-camera (controller, effects)

anvilkit-assets (glTF, animation, AssetServer) → anvilkit-core
anvilkit-audio (AudioSource, spatial) → anvilkit-ecs
```

## Per-Frame Lifecycle

```
┌─ Event Loop (winit) ──────────────────────────────────────────┐
│                                                                │
│  1. window_event / device_event                                │
│     ├─ forward_input()     → InputState (keys, mouse, scroll) │
│     └─ forward_device_input() → InputState.mouse_delta        │
│                                                                │
│  2. about_to_wait                                              │
│     ├─ tick() {                                                │
│     │    DeltaTime = wall_clock.elapsed()                      │
│     │    app.update() {                                        │
│     │       PreUpdate:  (reserved)                             │
│     │       Update:     input → physics → camera_fx → camera  │
│     │       PostUpdate: transform_propagation → render_extract │
│     │    }                                                     │
│     │    InputState.end_frame()                                │
│     │    window.request_redraw()                               │
│     │ }                                                        │
│     └─ post_update (game-specific: chunks, raycast, etc.)      │
│                                                                │
│  3. RedrawRequested                                            │
│     └─ render_frame() {                                        │
│          Shadow Pass ×3 (CSM cascades)                         │
│          HDR Scene Pass (PBR + MSAA 4×)                        │
│          SSAO Pass (half-res + blur)                           │
│          Bloom Pass (downsample ×5 + upsample ×4)             │
│          Tonemap Pass (HDR × AO + bloom → ACES → sRGB)        │
│          HUD Pass (lines, text, UI)                            │
│        }                                                       │
└────────────────────────────────────────────────────────────────┘
```

## ECS Schedule Phases

| Phase | Timing | Typical Systems |
|-------|--------|-----------------|
| `Startup` | Once at init | Resource setup, initial spawn |
| `PreUpdate` | Every frame, before Update | (reserved) |
| `Update` | Every frame, main logic | Input, physics, game logic, camera |
| `PostUpdate` | Every frame, after Update | Transform propagation, render extract |
| `Cleanup` | Every frame, last | (reserved) |

## Feature Flags

| Crate | Feature | What it enables |
|-------|---------|-----------------|
| `anvilkit-core` | `serde` | Serialize/Deserialize for Transform, GlobalTransform |
| `anvilkit-core` | `persistence` | SaveManager, Settings, WorldStorage (RON + file KV) |
| `anvilkit-core` | `bevy_ecs` | Component derives for math types |
| `anvilkit-ecs` | `serde` | Serialize for Parent, Children, scene serialization |
| `anvilkit-ecs` | `rapier` | rapier3d physics runtime integration |
| `anvilkit-assets` | `hot-reload` | File watching via `notify` crate |
| `anvilkit-assets` | `bevy_ecs` | Component derives for Skeleton, AnimationPlayer |

## Module Overview

### anvilkit-core

Foundation type library: math utilities (Vec3, Mat4, Transform, GlobalTransform, interpolation), time system (Time, Timer), error types (AnvilKitError), persistence system (SaveManager, Settings, WorldStorage).

### anvilkit-ecs

Application framework based on `bevy_ecs 0.14`: App lifecycle (new → add_plugins → add_systems → run), Plugin system, Schedule dispatching (Startup/PreUpdate/Update/PostUpdate), Transform hierarchy (Parent/Children/GlobalTransform propagation), Physics (AABB collision + rapier3d integration), Scene serialization (RON).

### anvilkit-render

Unified 2D/3D rendering pipeline:
- PBR material system (Cook-Torrance BRDF, 5 texture maps)
- HDR rendering (Rgba16Float) + ACES Filmic tone mapping
- Bloom (13-tap downsample + 9-tap tent upsample, 5 mip levels)
- SSAO (hemisphere sampling, half-res, depth-reconstructed normals)
- Cascade Shadow Maps (3 cascades, D2Array, PCF)
- IBL (BRDF LUT + environment irradiance)
- MSAA 4× anti-aliasing
- GPU skeletal animation (Linear Blend Skinning, 128 joints)
- Frustum culling + GPU instancing + batched draw sorting
- Sprite, Line, Particle, Text, UI renderers

### anvilkit-assets

Asset pipeline: glTF model loading (mesh, material, skeleton, animation), AssetServer (sync + async loading, deduplication), AssetHandle (Arc refcount), procedural mesh generation, file watching hot reload.

### anvilkit-audio

Audio system based on `rodio`: BGM and SFX playback, spatial audio (distance attenuation, panning), volume control (master/music/sfx).

### anvilkit-input

Input abstraction: InputState (keyboard/mouse pressed/just_pressed/released, mouse_delta, scroll_delta), ActionMap (logical action → physical key binding), RenderApp helpers (forward_input, forward_device_input, tick).

### anvilkit-camera

Camera system: first-person / third-person / free controllers, CameraEffects (head bob, landing shake, sprint FOV), gimbal lock protection, mouse sensitivity.

## Coordinate System

AnvilKit uses a **right-handed coordinate system** (consistent with wgpu/glam defaults):

- +X points right
- +Y points up
- -Z points into the screen (forward)

Projection uses `perspective_rh` and `look_at_rh`.
