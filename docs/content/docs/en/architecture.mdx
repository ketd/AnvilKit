## ECS Schedule Phases

| Phase | Timing | Typical Systems |
|-------|--------|-----------------|
| `Startup` | Once at init | Resource setup, initial spawn |
| `Main` | Every frame, first | (reserved for game-wide logic) |
| `PreUpdate` | Every frame, before Update | DeltaTime sync, parent-child hierarchy sync |
| `FixedUpdate` | 0-N per frame (accumulator) | Deterministic physics, fixed-rate gameplay |
| `Update` | Every frame, main logic | Input, physics, game logic, camera |
| `PostUpdate` | Every frame, after Update | Transform propagation, render extract |
| `Cleanup` | Every frame, last | (reserved) |

## Feature Flags

| Crate | Feature | What it enables |
|-------|---------|-----------------|
| `anvilkit-core` | `serde` | Serialize/Deserialize for Transform, GlobalTransform |
| `anvilkit-core` | `persistence` | SaveManager, Settings, WorldStorage (RON + file KV) |
| `anvilkit-core` | `bevy_ecs` | Component derives for math types |
| `anvilkit-ecs` | `serde` | Serialize for Parent, Children, scene serialization |
| `anvilkit-ecs` | `rapier` | rapier3d physics runtime integration |
| `anvilkit-assets` | `hot-reload` | File watching via `notify` crate |
| `anvilkit-assets` | `bevy_ecs` | Component derives for Skeleton, AnimationPlayer |

## Module Overview

### anvilkit-core

Foundation type library: math utilities (Vec3, Mat4, Transform, GlobalTransform, interpolation), time system (Time, Timer), error types (AnvilKitError with category-preserving `with_context`), persistence system (SaveManager, Settings, WorldStorage — all returning `Result<_, AnvilKitError>`).

### anvilkit-ecs

Application framework based on `bevy_ecs 0.14`: App lifecycle (new → add_plugins → add_systems → run), Plugin system with uniqueness checks, Schedule dispatching (Main/Startup/PreUpdate/Update/PostUpdate/Cleanup), AppExit resource for graceful shutdown, Transform hierarchy (Parent/Children/GlobalTransform propagation with auto-sync), Physics (scale-aware AABB collision + rapier3d integration), Scene serialization (RON), FixedUpdate schedule (accumulator-based deterministic timestep), Events system (bevy_ecs `Events<T>`/EventWriter/EventReader), `GameState<S>` state machine, SerializableRegistry for type-erased scene serialization, AutoInputPlugin and AutoDeltaTimePlugin for automatic frame management.

### anvilkit-render

Unified 2D/3D rendering pipeline:
- PBR material system (Cook-Torrance BRDF, 5 texture maps)
- HDR rendering (Rgba16Float) + ACES Filmic tone mapping
- Bloom (13-tap downsample + 9-tap tent upsample, 5 mip levels)
- SSAO (hemisphere sampling, half-res, depth-reconstructed normals)
- Cascade Shadow Maps (3 cascades, D2Array, PCF)
- IBL (BRDF LUT + environment irradiance)
- MSAA 4× anti-aliasing
- GPU skeletal animation (Linear Blend Skinning, 128 joints)
- Frustum culling + GPU instancing + batched draw sorting
- Sprite, Line, Particle, Text, UI renderers
- StandardMaterial component (PBR with builder pattern)
- PostProcessSettings (SSAO/DOF/MotionBlur/Bloom/ColorGrading)
- DemoApp scaffold
- Orthographic/Perspective projection enum, multi-camera priority
- PipelineCache/BindGroupCache
- Point/spot shadow system (ShadowAtlas)

### anvilkit-assets

Asset pipeline: glTF model loading (mesh, material, skeleton, animation), AssetServer (sync + async loading, deduplication), AssetHandle (Arc refcount), procedural mesh generation, file watching hot reload.

### anvilkit-audio

Audio system based on `rodio`: AudioSource component with ECS integration, AudioBus (master/music/sfx/voice volume mixing), spatial audio (distance attenuation via AudioListener), looping and pitch control.

### anvilkit-input

Input abstraction: InputState (keyboard/mouse pressed/just_pressed/released, mouse_delta, scroll_delta), GamepadState (16 buttons, 6 axes), ActionMap (logical action → physical binding), ActionId (zero-allocation lookups), AxisBinding (keyboard/gamepad analog axes).

### anvilkit-camera

Camera system: first-person / third-person / free controllers, CameraEffects (head bob, landing shake, sprint FOV), gimbal lock protection, mouse sensitivity.

## Coordinate System

AnvilKit uses a **left-handed coordinate system**:

- +X points right
- +Y points up
- +Z points into the screen (forward)

Projection uses `perspective_lh` and `look_at_lh`.
