## Prerequisites

- Rust 1.70 or higher
- GPU with Vulkan / Metal / DX12 support

## Using the CLI

AnvilKit provides a CLI tool for quick project scaffolding:

```bash
# Install the CLI
cargo install --path tools/anvilkit-cli

# Create a new project
anvilkit new my_game --template 3d-basic
```

Available templates:

| Template | Description |
|----------|-------------|
| `empty` | Empty project with basic structure only |
| `3d-basic` | Basic 3D scene with camera and lighting |
| `topdown` | Top-down 2D game template |
| `first-person` | First-person 3D game template |

## Manual Setup

If you prefer manual setup, add dependencies in your workspace `Cargo.toml`:

```toml
[dependencies]
anvilkit = { path = "../../crates/anvilkit" }
```

## Minimal Example

Since v0.3, AnvilKit uses the `AnvilKitApp::run()` + `GameCallbacks` pattern.
The app runner handles the event loop, input forwarding, and frame lifecycle
automatically — you only implement the callbacks your game needs.

```rust
use anvilkit::prelude::*;

struct MyGame;

impl GameCallbacks for MyGame {
    fn init(&mut self, ctx: &mut GameContext) {
        ctx.app.add_systems(AnvilKitSchedule::Update, hello_system);
    }
}

fn main() {
    AnvilKitApp::run(
        GameConfig::new("My Game").with_size(1280, 720),
        MyGame,
    );
}

fn hello_system() {
    println!("Hello from AnvilKit!");
}
```

## Running Your Project

```bash
# Using the CLI
anvilkit run my_game

# Or directly with Cargo
cargo run -p my_game
```
