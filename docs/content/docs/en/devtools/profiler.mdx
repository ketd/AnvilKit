## Overview

`FrameProfiler` provides CPU-side frame timing and per-section profiling for the game loop. It is feature-gated behind the `"debug"` feature flag. When the feature is not enabled, all methods compile to no-ops with zero runtime overhead.

The profiler is provided by the `anvilkit-ecs` crate:

```rust
use anvilkit_ecs::profiler::FrameProfiler;
```

`FrameProfiler` is an ECS `Resource` that maintains a rolling history window of frame times (default: 300 frames). It also tracks per-named-section timings and supports external GPU timing data.

## FrameProfiler API

| Method | Signature | Description |
|--------|-----------|-------------|
| `new` | `() -> Self` | Create a new profiler with default settings (300-frame history) |
| `begin_frame` | `(&mut self)` | Record the start of a new frame |
| `end_frame` | `(&mut self)` | Record the end of the current frame; computes elapsed time and pushes to history |
| `begin_section` | `(&mut self, name: &str)` | Begin a named timing section within the current frame |
| `end_section` | `(&mut self, name: &str)` | End a named timing section; stores elapsed time in per-section history |
| `fps` | `(&self) -> f64` | Average frames per second over the history window |
| `avg_frame_time_ms` | `(&self) -> f64` | Average frame time in milliseconds over the history window |
| `avg_section_time_ms` | `(&self, name: &str) -> Option<f64>` | Average time in milliseconds for a named section, or `None` if no data recorded |
| `min_frame_time_ms` | `(&self) -> f64` | Minimum frame time in milliseconds over the history window |
| `max_frame_time_ms` | `(&self) -> f64` | Maximum frame time in milliseconds over the history window |
| `percentile_frame_time_ms` | `(&self, p: f64) -> f64` | Percentile frame time in milliseconds (e.g., `p = 0.99` for 99th percentile) |

### Public Fields (debug feature only)

| Field | Type | Description |
|-------|------|-------------|
| `frame_times` | `VecDeque<f64>` | Rolling history of frame times in seconds |
| `gpu_times` | `VecDeque<f64>` | Rolling history of GPU times in seconds (filled externally) |
| `section_times` | `HashMap<String, VecDeque<f64>>` | Per-named-section timing histories in seconds |
| `frame_count` | `u64` | Total number of frames recorded |
| `max_history` | `usize` | Maximum number of frames kept in history (default: 300) |

## Usage Example

```rust
use anvilkit_ecs::profiler::FrameProfiler;

let mut profiler = FrameProfiler::new();

// In the game loop:
profiler.begin_frame();

profiler.begin_section("physics");
// ... physics simulation ...
profiler.end_section("physics");

profiler.begin_section("render");
// ... rendering ...
profiler.end_section("render");

profiler.end_frame();

// Query statistics:
println!("FPS: {:.1}", profiler.fps());
println!("Avg frame: {:.2} ms", profiler.avg_frame_time_ms());
println!("Min frame: {:.2} ms", profiler.min_frame_time_ms());
println!("Max frame: {:.2} ms", profiler.max_frame_time_ms());
println!("99th percentile: {:.2} ms", profiler.percentile_frame_time_ms(0.99));

if let Some(physics_ms) = profiler.avg_section_time_ms("physics") {
    println!("Physics avg: {:.2} ms", physics_ms);
}
```

When the `"debug"` feature is disabled, all the above calls compile to no-ops. `fps()`, `avg_frame_time_ms()`, `min_frame_time_ms()`, `max_frame_time_ms()`, and `percentile_frame_time_ms()` return `0.0`, and `avg_section_time_ms()` returns `None`.
