## UI System

Retained-mode UI framework with flexbox layout, event system, widget constructors, and interactive controls. The `anvilkit-ui` crate owns the data model, layout, and event logic. GPU rendering is delegated to `anvilkit-render`'s `UiRenderer` + `UiVertex`.

**Layout**: Powered by `taffy` (full CSS Flexbox subset). `UiLayoutEngine::compute_layout()` takes `UiNode` entities and a container size, outputs `[x, y, w, h]` computed rectangles.

**Rendering**: `UiRenderer` (in `anvilkit-render`) draws rounded rectangles with background color, border, and text in a single draw call with alpha blending over the 3D scene.

## UiPlugin

`UiPlugin` registers the layout and event systems into the ECS schedule:

```rust
use anvilkit_ui::prelude::*;

app.add_plugins(UiPlugin);
// This registers:
//   - layout_system in PostUpdate (computes flexbox layout each frame)
//   - event_system in Update (hit testing, hover/click/focus dispatch)
```

## UiTheme

`UiTheme` is a resource that provides default styling values for all widgets:

```rust
use anvilkit_ui::prelude::*;

let theme = UiTheme::default();
// theme.colors — background, foreground, accent, border, hover, pressed
// theme.fonts — family, size, line_height
// theme.spacing — padding, margin, gap defaults
// theme.borders — width, radius
```

Insert a custom theme as a resource to override defaults globally.

## Event System

| Event | Trigger |
|-------|---------|
| `HoverEnter` | Mouse enters node bounds |
| `HoverLeave` | Mouse leaves node bounds |
| `Click` | Mouse button pressed within node |
| `UiChangeEvent` | Widget value changed (checkbox, slider, text input) |

```rust
use anvilkit_ui::prelude::*;

// Create widgets
let button = Widget::button("Start Game");
let panel = Widget::panel();
let label = Widget::label("Score: 100");
let row = Widget::row();    // horizontal layout
let col = Widget::column();  // vertical layout

// Check events
let events = world.resource::<UiEvents>();
if events.was_clicked(button_entity) {
    // Handle button click
}
if events.is_hovered(button_entity) {
    // Show hover effect
}
```

## Focus Management

Tab navigation cycles focus through focusable widgets. Each interactive widget carries a `UiInteraction` component:

| State | Meaning |
|-------|---------|
| `UiInteraction::None` | Default, no interaction |
| `UiInteraction::Hovered` | Mouse is over the widget |
| `UiInteraction::Pressed` | Mouse button held down on the widget |
| `UiInteraction::Focused` | Widget has keyboard focus (Tab or click) |

```rust
use anvilkit_ui::prelude::*;

// Query focused widget
fn my_system(query: Query<(Entity, &UiInteraction)>) {
    for (entity, interaction) in &query {
        if *interaction == UiInteraction::Focused {
            // This widget has keyboard focus
        }
    }
}
```

**UiStyle fields**: `flex_direction`, `justify_content`, `align_items`, `width`, `height`, `min_width`, `min_height`, `max_width`, `max_height`, `padding[4]`, `margin[4]`, `gap`, `flex_grow`, `flex_shrink`.

## Interactive Widgets

### TextInput

Editable text field with cursor and keyboard input support.

```rust
use anvilkit_ui::prelude::*;

let input = Widget::text_input("Placeholder...");
// Supports: cursor movement (left/right/home/end), character input,
// backspace/delete, text selection, and UiChangeEvent on edit.
```

### Slider

Draggable handle for selecting a numeric value within a range.

```rust
use anvilkit_ui::prelude::*;

let slider = Widget::slider(0.0, 100.0, 50.0); // min, max, initial
// Drag the handle or click the track to update the value.
// Emits UiChangeEvent with the new value.
```

### Checkbox

Toggle control that emits `UiChangeEvent` on click.

```rust
use anvilkit_ui::prelude::*;

let checkbox = Widget::checkbox(false); // initial checked state
// Click toggles between checked/unchecked.
// Emits UiChangeEvent with the new bool value.
```

### Dropdown

Selection list that expands on click.

```rust
use anvilkit_ui::prelude::*;

let dropdown = Widget::dropdown(&["Option A", "Option B", "Option C"], 0);
// Click opens the selection list; selecting an item closes it.
// Emits UiChangeEvent with the selected index.
```

### ScrollView

Scrollable container for content that exceeds the visible area.

```rust
use anvilkit_ui::prelude::*;

let scroll = Widget::scroll_view();
// Supports mousewheel scrolling and drag-to-scroll.
// Child widgets are clipped to the scroll view bounds.
```

## Custom Render Pipeline

For games that bypass the built-in PBR pipeline (e.g., voxel games), use `RenderApp` in low-level mode with the helper API:

```rust
use anvilkit_render::window::RenderApp;

impl ApplicationHandler for MyGame {
    fn window_event(&mut self, el, wid, ev) {
        // Game-specific key handling here...

        // Engine handles input forwarding (keyboard, mouse, cursor, scroll)
        RenderApp::forward_input(&mut self.app, &ev);
        self.render_app.window_event(el, wid, ev);
    }

    fn device_event(&mut self, el, did, ev) {
        // Engine forwards raw mouse motion → InputState.mouse_delta
        RenderApp::forward_device_input(&mut self.app, &ev);
        self.render_app.device_event(el, did, ev);
    }

    fn about_to_wait(&mut self, _el) {
        // Engine tick: DeltaTime → app.update() → InputState.end_frame() → request_redraw
        self.render_app.tick(&mut self.app);
        // Your post-update logic here...
    }
}
```
