## Transform Hierarchy

`TransformPlugin` registers `parent_child_sync_system` in **PreUpdate** and `sync_simple_transforms` + `propagate_transforms` in **PostUpdate** under `AnvilKitSystemSet::Transform`. Transform propagation runs every frame for all root entities, ensuring child-only `Transform` changes are always reflected in `GlobalTransform`.

### Core Components

| Component | Description |
|-----------|-------------|
| `Transform` | Local transform relative to parent (translation, rotation, scale) |
| `GlobalTransform` | World-space transform matrix, computed by the propagation system |
| `Parent(Entity)` | Reference to parent entity |
| `Children` | List of child entities |

### Spawning a Parent-Child Hierarchy

```rust
use anvilkit_ecs::prelude::*;
use glam::Vec3;

fn setup_hierarchy(mut commands: Commands) {
    // Spawn parent at world position (10, 0, 0)
    let parent = commands.spawn((
        Transform::from_xyz(10.0, 0.0, 0.0),
        GlobalTransform::default(),
        Children::empty(),
    )).id();

    // Spawn child offset (5, 0, 0) from parent
    let child = commands.spawn((
        Transform::from_xyz(5.0, 0.0, 0.0),
        GlobalTransform::default(),
        Parent::new(parent),
    )).id();

    // After propagation, child's GlobalTransform.translation() == (15, 0, 0)
}
```

### TransformHierarchy Utilities

| Method | Signature | Description |
|--------|-----------|-------------|
| `set_parent` | `(Commands, child: Entity, parent: Entity)` | Establish parent-child relationship (Children auto-synced by `parent_child_sync_system`) |
| `remove_parent` | `(Commands, child: Entity)` | Detach child from parent |
| `get_ancestors` | `(&World, Entity) -> Vec<Entity>` | Walk up the hierarchy to root (cycle-safe) |
| `get_descendants` | `(&World, Entity) -> Vec<Entity>` | Recursively collect all descendants (depth limit: 128) |

## Scene Serialization (feature `"serde"`)

`SceneSerializer` saves/loads entities marked with the `Serializable` component to RON files.

| Type | Description |
|------|-------------|
| `Serializable` | Marker component -- only entities with this are serialized |
| `SceneSerializer::save` | `(&mut World, path: &str) -> Result<usize, AnvilKitError>` -- returns entity count |
| `SceneSerializer::load` | `(&mut World, path: &str) -> Result<usize, AnvilKitError>` -- spawns entities, returns count |

```rust
use anvilkit_ecs::prelude::*;
use anvilkit_ecs::scene::{SceneSerializer, Serializable};

// Mark entities for serialization
fn setup(mut commands: Commands) {
    commands.spawn((
        Transform::from_xyz(1.0, 2.0, 3.0),
        GlobalTransform::default(),
        Serializable,
    ));
}

// Save the scene
fn save_game(world: &mut World) {
    let count = SceneSerializer::save(world, "scene.ron").unwrap();
    log::info!("Saved {} entities", count);
}

// Load into a fresh world
fn load_game(world: &mut World) {
    let count = SceneSerializer::load(world, "scene.ron").unwrap();
    log::info!("Loaded {} entities", count);
}
```

The RON file stores a `SerializedScene { version: u32, entities: Vec<SerializedEntity> }` where each entity captures its `Transform` as `TransformData { translation: [f32; 3], rotation: [f32; 4], scale: [f32; 3] }`.

### Hierarchy Preservation

Scene serialization preserves parent-child relationships:

- `parent_index`: References the parent entity's position in the `entities` array
- `name` / `tag`: Captured from `Name` and `Tag` components
- `custom_data`: `HashMap<String, String>` for user extensions

The serializer uses a two-pass approach: first pass creates all entities, second pass restores `Parent`/`Children` components.

## SerializableRegistry

Register custom component types for scene serialization:

```rust
use anvilkit_ecs::scene::SerializableRegistry;

let mut app = App::new();
app.register_serializable::<Health>("Health");
app.register_serializable::<Inventory>("Inventory");

// Or directly:
let mut registry = SerializableRegistry::new();
registry.register::<Health>("Health");
assert!(registry.is_registered("Health"));
```
