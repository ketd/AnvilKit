`anvilkit-render` is AnvilKit's core rendering module, providing unified 2D/3D rendering capabilities based on `wgpu 0.19`.

## Render Pipeline

The engine runs a multi-pass HDR pipeline per frame:

```
Pass 0  CSM Shadow (×3 cascades)     → D2Array depth texture
Pass 1  HDR Scene (PBR + MSAA 4×)    → Rgba16Float RT (resolve)
Pass 2  SSAO (half-res + blur)        → R8Unorm AO texture
Pass 3  Bloom (downsample ×5 + upsample ×4) → Rgba16Float mip[0]
Pass 4  Tonemap (HDR × AO + Bloom → ACES Filmic → sRGB) → Swapchain
```

All passes share a single `CommandEncoder` per frame for minimal CPU overhead.

## Bloom

Physically-based bloom extracted from HDR values above a brightness threshold.

**Pipeline**: 13-tap weighted downsample (prevents firefly artifacts) → progressive 5-level mip chain → 9-tap tent filter upsample with additive blending → composite in tonemap shader.

**Configuration** (`BloomSettings` ECS resource):

| Parameter | Default | Description |
|-----------|---------|-------------|
| `enabled` | `true` | Toggle bloom on/off (zero GPU cost when off) |
| `threshold` | `1.0` | HDR brightness threshold for bloom extraction |
| `knee` | `0.1` | Soft knee range for smooth threshold transition |
| `intensity` | `0.3` | Bloom contribution multiplier |
| `mip_count` | `5` | Number of downsample levels |

```rust
use anvilkit_render::renderer::bloom::BloomSettings;

// Runtime adjustment
app.insert_resource(BloomSettings {
    threshold: 0.8,
    intensity: 0.5,
    ..Default::default()
});
```

**Files**: `bloom.rs`, `bloom_downsample.wgsl`, `bloom_upsample.wgsl`

## SSAO

Screen-space ambient occlusion using hemisphere kernel sampling with depth-reconstructed normals.

**Pipeline**: Depth buffer → reconstruct view-space normals (cross-product of screen-space derivatives) → hemisphere sampling with 4×4 noise texture rotation → half-resolution AO → 4×4 box blur → multiply into scene in tonemap.

**Configuration** (`SsaoSettings` ECS resource):

| Parameter | Default | Description |
|-----------|---------|-------------|
| `enabled` | `true` | Toggle SSAO on/off |
| `quality` | `Medium` | Sample count: `Low`=16, `Medium`=32, `High`=64 |
| `radius` | `0.5` | Sampling hemisphere radius in view-space units |
| `bias` | `0.025` | Depth bias to prevent self-occlusion on flat surfaces |
| `intensity` | `1.0` | AO darkness multiplier (higher = stronger occlusion) |

```rust
use anvilkit_render::renderer::ssao::{SsaoSettings, SsaoQuality};

app.insert_resource(SsaoSettings {
    quality: SsaoQuality::High,
    radius: 0.3,
    ..Default::default()
});
```

**Files**: `ssao.rs`, `ssao.wgsl`, `ssao_blur.wgsl`

## Cascade Shadow Maps

3-level cascade shadow maps replacing the single-level shadow map for distance-dependent shadow quality.

**How it works**: The camera frustum is split into 3 slices at 10%, 30%, and 100% of the far plane. Each slice gets its own orthographic projection tightly fit around the frustum corners. The PBR fragment shader selects the tightest cascade per-fragment based on view-space Z depth.

| Cascade | Coverage | Quality |
|---------|----------|---------|
| 0 | Near (0–10% of far) | Highest resolution |
| 1 | Mid (10–30% of far) | Medium resolution |
| 2 | Far (30–100% of far) | Lowest resolution |

**GPU Layout** (`PbrSceneUniform`):

| Field | Type | Description |
|-------|------|-------------|
| `cascade_view_projs` | `[mat4×4; 3]` | Per-cascade light-space view-projection matrices |
| `cascade_splits` | `[f32; 4]` | `[split0, split1, split2, shadow_texel_size]` |
| `emissive_factor.w` | `f32` | Cascade count (3.0) |

Shadow map texture: `Depth32Float`, `2048×2048`, `depth_or_array_layers = 3` (`D2Array`).

**Files**: `events.rs` (`compute_cascade_matrices`), `pbr.wgsl` (`calculate_shadow`)

## PBR Material System

Cook-Torrance specular BRDF with GGX distribution, Smith geometry, Fresnel-Schlick.

**Bind Groups**:

| Group | Contents |
|-------|----------|
| 0 | Scene uniform (992 bytes): model, view_proj, normal_matrix, camera_pos, lights[8], cascade matrices |
| 1 | Material textures: base color, normal map, metallic-roughness, AO, emissive (5 textures + 5 samplers) |
| 2 | IBL + Shadow: BRDF LUT, linear sampler, CSM shadow map array, comparison sampler |
| 3 | Joint matrices (skinned only): `storage<array<mat4×4, 128>>` |

## Skeletal Animation

GPU Linear Blend Skinning with up to 128 joints per skeleton.

**Vertex format** (`SkinnedVertex`, 72 bytes):

| Attribute | Type | Location |
|-----------|------|----------|
| `position` | `vec3<f32>` | 0 |
| `normal` | `vec3<f32>` | 1 |
| `texcoord` | `vec2<f32>` | 2 |
| `tangent` | `vec4<f32>` | 3 |
| `joint_indices` | `u16×4` | 4 |
| `joint_weights` | `f32×4` | 5 |

**Skinning equation** (in `skinned_pbr.wgsl`):
```
skinned_pos = Σ(weight[i] × joints[index[i]] × position)  for i in 0..4
```

**Runtime flow**:
1. `AnimationPlayer::advance(dt)` updates `current_time`
2. `compute_bone_matrices(skeleton, player)` computes per-joint `global × inverse_bind`
3. Upload `BoneMatrices` to storage buffer (bind group 3)
4. Vertex shader applies LBS before model/view-proj transform

## UI System

Flexbox-based UI with GPU-accelerated rectangle rendering, event system, and widget constructors.

**Layout**: Powered by `taffy` (full CSS Flexbox subset). `UiLayoutEngine::compute_layout()` takes `UiNode` entities and a container size, outputs `[x, y, w, h]` computed rectangles.

**Rendering**: `UiRenderer` draws rounded rectangles with background color, border, and text in a single draw call with alpha blending over the 3D scene.

**Event System**:

| Event | Trigger |
|-------|---------|
| `HoverEnter` | Mouse enters node bounds |
| `HoverLeave` | Mouse leaves node bounds |
| `Click` | Mouse button pressed within node |

```rust
use anvilkit_render::renderer::ui::{UiEvents, Widget, UiNode};

// Create widgets
let button = Widget::button("Start Game");
let panel = Widget::panel();
let label = Widget::label("Score: 100");
let row = Widget::row();    // horizontal layout
let col = Widget::column();  // vertical layout

// Check events
let events = world.resource::<UiEvents>();
if events.was_clicked(button_entity) {
    // Handle button click
}
if events.is_hovered(button_entity) {
    // Show hover effect
}
```

**UiStyle fields**: `flex_direction`, `justify_content`, `align_items`, `width`, `height`, `min_width`, `min_height`, `max_width`, `max_height`, `padding[4]`, `margin[4]`, `gap`, `flex_grow`, `flex_shrink`.

## Custom Render Pipeline

For games that bypass the built-in PBR pipeline (e.g., voxel games), use `RenderApp` in low-level mode with the helper API:

```rust
use anvilkit_render::window::RenderApp;

impl ApplicationHandler for MyGame {
    fn window_event(&mut self, el, wid, ev) {
        // Game-specific key handling here...

        // Engine handles input forwarding (keyboard, mouse, cursor, scroll)
        RenderApp::forward_input(&mut self.app, &ev);
        self.render_app.window_event(el, wid, ev);
    }

    fn device_event(&mut self, el, did, ev) {
        // Engine forwards raw mouse motion → InputState.mouse_delta
        RenderApp::forward_device_input(&mut self.app, &ev);
        self.render_app.device_event(el, did, ev);
    }

    fn about_to_wait(&mut self, _el) {
        // Engine tick: DeltaTime → app.update() → InputState.end_frame() → request_redraw
        self.render_app.tick(&mut self.app);
        // Your post-update logic here...
    }
}
```

## Resource Management

`RenderAssets` manages GPU resources through a Handle system:

```rust
let handle = render_assets.upload_texture(device, queue, &texture_data);
let mesh_handle = render_assets.upload_mesh(device, &mesh_data);

if let Some(texture) = render_assets.get_texture(handle) {
    // Use texture
}
```

`BufferPool` provides efficient GPU buffer reuse:

```rust
let (buffer, capacity) = pool.acquire(device, needed_size, BufferUsages::VERTEX);
// ... use buffer ...
pool.release(buffer, capacity, BufferUsages::VERTEX);
```
