`anvilkit-ecs` provides application lifecycle management, a Plugin system, System scheduling, transform hierarchy, built-in physics, and scene serialization, all built on top of `bevy_ecs`.

## App Lifecycle

`App` is the application entry point. The typical setup flow is:

```
App::new() → add_plugins() → insert_resource() → add_systems() → run() / update()
```

```rust
use anvilkit_ecs::prelude::*;
use anvilkit_ecs::schedule::AnvilKitSchedule;

let mut app = App::new();

// 1. Register plugins (order matters — dependencies first)
app.add_plugins(AnvilKitEcsPlugin);
app.add_plugins(TransformPlugin);
app.add_plugins(PhysicsPlugin);

// 2. Insert global resources
app.insert_resource(MyConfig { difficulty: 3 });
app.init_resource::<Score>();          // uses Default

// 3. Register systems into schedules
app.add_systems(AnvilKitSchedule::Startup, setup_scene);
app.add_systems(AnvilKitSchedule::Update, (player_input, enemy_ai));
app.add_systems(AnvilKitSchedule::PostUpdate, cleanup);

// 4a. Blocking main loop (calls update() in a loop)
app.run();

// 4b. Or drive updates manually
for _ in 0..60 {
    app.update(); // one frame
}
```

### App API

| Method | Signature | Description |
|--------|-----------|-------------|
| `new` | `() -> Self` | Create an App with all engine schedules pre-registered |
| `add_plugins` | `<P: Plugin>(P) -> &mut Self` | Register a plugin (unique plugins are deduplicated) |
| `add_systems` | `(impl ScheduleLabel, impl IntoSystemConfigs) -> &mut Self` | Add systems to a schedule |
| `insert_resource` | `<R: Resource>(R) -> &mut Self` | Insert a typed global resource |
| `init_resource` | `<R: Resource + FromWorld>() -> &mut Self` | Insert a resource using its Default/FromWorld impl |
| `run` | `(&mut self)` | Blocking main loop until `exit()` is called |
| `update` | `(&mut self)` | Execute one frame: Startup (first call only), then PreUpdate -> Update -> PostUpdate |
| `exit` | `(&mut self)` | Signal the app to stop after the current frame |

## Plugin

Plugins encapsulate reusable engine functionality. Implement the `Plugin` trait:

```rust
pub struct PhysicsPlugin;

impl Plugin for PhysicsPlugin {
    fn build(&self, app: &mut App) {
        app.init_resource::<DeltaTime>()
           .init_resource::<CollisionEvents>()
           .add_systems(AnvilKitSchedule::Update, (
               velocity_integration_system,
               collision_detection_system.after(velocity_integration_system),
           ));
    }
}
```

Unique plugins (default) are registered at most once; duplicate `add_plugins` calls are silently skipped.

## Schedule Phases

`App::update()` runs schedules in the following order every frame:

| Schedule | Variant | Runs | Typical use |
|----------|---------|------|-------------|
| **Startup** | `AnvilKitSchedule::Startup` | Once, on first `update()` | Scene setup, asset loading |
| **PreUpdate** | `AnvilKitSchedule::PreUpdate` | Every frame | Input polling, time update |
| **Update** | `AnvilKitSchedule::Update` | Every frame | Game logic, AI, physics |
| **PostUpdate** | `AnvilKitSchedule::PostUpdate` | Every frame | Transform propagation, render prep |
| **Cleanup** | `AnvilKitSchedule::Cleanup` | Every frame | Temp data cleanup, diagnostics |

## SystemSet

Within a schedule, systems can be grouped by `AnvilKitSystemSet` for ordering:

| SystemSet | Purpose |
|-----------|---------|
| `Input` | Keyboard, mouse, gamepad processing |
| `Time` | Time/timer updates, frame-rate tracking |
| `Physics` | Velocity integration, collision detection, physics step |
| `GameLogic` | AI, state machines, game rules |
| `Transform` | Hierarchy propagation, coordinate sync |
| `Render` | Visibility culling, material updates, draw prep |
| `Audio` | Sound playback, volume control |
| `UI` | Layout, event handling, widget updates |
| `Network` | Replication, serialization, connection management |
| `Debug` | Gizmos, performance overlay, logging |

```rust
app.add_systems(AnvilKitSchedule::Update, (
    apply_gravity.in_set(AnvilKitSystemSet::Physics),
    check_collisions.in_set(AnvilKitSystemSet::Physics)
        .after(apply_gravity),
    handle_input.in_set(AnvilKitSystemSet::Input),
));
```

## System

Systems are plain functions whose parameters declare data access:

```rust
fn move_system(
    time: Res<Time>,
    mut query: Query<(&Velocity, &mut Transform)>,
) {
    let dt = time.delta_seconds();
    for (vel, mut transform) in &mut query {
        transform.translation += vel.linear * dt;
    }
}

fn collision_handler(events: Res<CollisionEvents>) {
    for event in events.iter() {
        println!("Collision: {:?} <-> {:?}", event.a, event.b);
    }
}
```

## Transform Hierarchy

`TransformPlugin` registers `sync_simple_transforms` and `propagate_transforms` in **PostUpdate** under `AnvilKitSystemSet::Transform`.

### Core Components

| Component | Description |
|-----------|-------------|
| `Transform` | Local transform relative to parent (translation, rotation, scale) |
| `GlobalTransform` | World-space transform matrix, computed by the propagation system |
| `Parent(Entity)` | Reference to parent entity |
| `Children` | List of child entities |

### Spawning a Parent-Child Hierarchy

```rust
use anvilkit_ecs::prelude::*;
use glam::Vec3;

fn setup_hierarchy(mut commands: Commands) {
    // Spawn parent at world position (10, 0, 0)
    let parent = commands.spawn((
        Transform::from_xyz(10.0, 0.0, 0.0),
        GlobalTransform::default(),
        Children::empty(),
    )).id();

    // Spawn child offset (5, 0, 0) from parent
    let child = commands.spawn((
        Transform::from_xyz(5.0, 0.0, 0.0),
        GlobalTransform::default(),
        Parent::new(parent),
    )).id();

    // After propagation, child's GlobalTransform.translation() == (15, 0, 0)
}
```

### TransformHierarchy Utilities

| Method | Signature | Description |
|--------|-----------|-------------|
| `set_parent` | `(Commands, child: Entity, parent: Entity)` | Establish parent-child relationship |
| `remove_parent` | `(Commands, child: Entity)` | Detach child from parent |
| `get_ancestors` | `(&World, Entity) -> Vec<Entity>` | Walk up the hierarchy to root |
| `get_descendants` | `(&World, Entity) -> Vec<Entity>` | Recursively collect all descendants |

## Physics

### Built-in AABB Physics (default)

`PhysicsPlugin` registers velocity integration and N-squared AABB collision detection in `Update`.

| Type | Kind | Description |
|------|------|-------------|
| `DeltaTime` | Resource | Frame delta in seconds (default 1/60) |
| `Velocity` | Component | `linear: Vec3`, `angular: Vec3` |
| `AabbCollider` | Component | `half_extents: Vec3` |
| `CollisionEvents` | Resource | `Vec<CollisionEvent>` populated each frame |
| `CollisionEvent` | Struct | `a: Entity`, `b: Entity` |

| System | Description |
|--------|-------------|
| `velocity_integration_system` | `transform.translation += velocity.linear * dt` |
| `collision_detection_system` | N-squared AABB overlap test, writes `CollisionEvents` |

```rust
use anvilkit_ecs::prelude::*;
use anvilkit_ecs::physics::*;

fn setup_physics(mut commands: Commands) {
    // Dynamic moving box
    commands.spawn((
        Transform::from_xyz(0.0, 10.0, 0.0),
        Velocity::linear(Vec3::new(0.0, -5.0, 0.0)),
        AabbCollider::cube(0.5),
    ));

    // Static ground
    commands.spawn((
        Transform::from_xyz(0.0, 0.0, 0.0),
        AabbCollider::new(Vec3::new(50.0, 0.1, 50.0)),
    ));
}

fn read_collisions(events: Res<CollisionEvents>) {
    for event in events.iter() {
        log::info!("Hit: {:?} <-> {:?}", event.a, event.b);
    }
}
```

### Rapier Integration (feature `"rapier"`)

Enable the `rapier` feature for full rigid-body simulation with `rapier3d`.

`RapierPhysicsPlugin` registers four systems in `Update`:

| System | Order | Description |
|--------|-------|-------------|
| `sync_to_rapier_system` | 1st | Creates rapier bodies from ECS `RigidBody`+`Collider`+`Transform` |
| `step_physics_system` | 2nd | Steps the rapier simulation one tick |
| `sync_from_rapier_system` | 3rd | Writes rapier positions back to ECS `Transform` |
| `extract_collision_events_system` | 3rd | Populates `CollisionEvents` from rapier narrow-phase |

| Type | Description |
|------|-------------|
| `RapierContext` | Resource holding all rapier state (rigid body set, collider set, pipeline, etc.) |
| `RigidBody` | Component: `body_type`, `mass`, `gravity_scale`, `is_sensor`, damping |
| `Collider` | Component: `shape` (Sphere/Cuboid/Capsule/TriMesh), `friction`, `restitution` |
| `RaycastHit` | Result struct: `entity`, `point`, `normal`, `distance` |

```rust
use anvilkit_ecs::prelude::*;
use anvilkit_ecs::physics::*;

fn setup_rapier(mut commands: Commands) {
    // Dynamic sphere
    commands.spawn((
        Transform::from_xyz(0.0, 10.0, 0.0),
        RigidBody::dynamic(),
        Collider::sphere(0.5),
        Velocity::default(),
    ));

    // Static ground
    commands.spawn((
        Transform::default(),
        RigidBody::fixed(),
        Collider::cuboid(Vec3::new(50.0, 0.1, 50.0)),
    ));
}

// Raycast query
fn shoot_ray(ctx: Res<RapierContext>) {
    if let Some(hit) = ctx.raycast(
        Vec3::new(0.0, 5.0, 0.0), // origin
        Vec3::NEG_Y,               // direction
        100.0,                     // max distance
    ) {
        log::info!(
            "Hit entity {:?} at {} (distance {})",
            hit.entity, hit.point, hit.distance
        );
    }
}
```

## Scene Serialization (feature `"serde"`)

`SceneSerializer` saves/loads entities marked with the `Serializable` component to RON files.

| Type | Description |
|------|-------------|
| `Serializable` | Marker component -- only entities with this are serialized |
| `SceneSerializer::save` | `(&mut World, path: &str) -> Result<usize, String>` -- returns entity count |
| `SceneSerializer::load` | `(&mut World, path: &str) -> Result<usize, String>` -- spawns entities, returns count |

```rust
use anvilkit_ecs::prelude::*;
use anvilkit_ecs::scene::{SceneSerializer, Serializable};

// Mark entities for serialization
fn setup(mut commands: Commands) {
    commands.spawn((
        Transform::from_xyz(1.0, 2.0, 3.0),
        GlobalTransform::default(),
        Serializable,
    ));
}

// Save the scene
fn save_game(world: &mut World) {
    let count = SceneSerializer::save(world, "scene.ron").unwrap();
    log::info!("Saved {} entities", count);
}

// Load into a fresh world
fn load_game(world: &mut World) {
    let count = SceneSerializer::load(world, "scene.ron").unwrap();
    log::info!("Loaded {} entities", count);
}
```

The RON file stores a `SerializedScene { version: u32, entities: Vec<SerializedEntity> }` where each entity captures its `Transform` as `TransformData { translation: [f32; 3], rotation: [f32; 4], scale: [f32; 3] }`.
