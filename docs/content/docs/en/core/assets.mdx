`anvilkit-assets` provides asset loading and management, including glTF models, skeletal animation, and procedural mesh generation.

## glTF Loading

```rust
use anvilkit_assets::gltf_loader::GltfLoader;

let scene = GltfLoader::load("assets/models/character.glb")?;

for mesh in &scene.meshes {
    println!("Mesh: {} vertices, {} triangles",
        mesh.positions.len(),
        mesh.indices.len() / 3
    );
}
```

## Skeletal Animation

Three interpolation modes are supported:

| Mode | Description |
|------|-------------|
| `Linear` | Linear interpolation (lerp for translation/scale, slerp for rotation) |
| `Step` | Step function, no transition |
| `CubicSpline` | Cubic spline interpolation for smoother curves |

```rust
use anvilkit_assets::animation::*;

let mut player = AnimationPlayer::new();
player.play(animation_clip, true); // loop = true
player.update(dt);
let joint_transforms = player.joint_transforms();
```

## Procedural Mesh Generation

`MeshData` provides factory methods for common primitives:

```rust
use anvilkit_assets::mesh::MeshData;

let cube = MeshData::generate_box(1.0);       // 24 vertices, 36 indices
let plane = MeshData::generate_plane(10.0);    // 4 vertices, 6 indices (XZ plane, +Y normal)
let sphere = MeshData::generate_sphere(1.0, 32, 16); // UV sphere

// Convert to GPU-ready format
let pbr_vertices = cube.to_pbr_vertices();
```

| Method | Parameters | Description |
|--------|-----------|-------------|
| `generate_box(size)` | `f32` | Centered cube, 6 faces with independent normals |
| `generate_plane(size)` | `f32` | XZ plane centered at origin, +Y normal |
| `generate_sphere(radius, segments, rings)` | `f32, u32, u32` | UV sphere |
| `to_pbr_vertices()` | — | Convert to interleaved `InterleavedPbrVertex` (48 bytes/vertex) |

### StandardMaterial

For high-level PBR rendering, use `StandardMaterial` (in `anvilkit-render`):

```rust
use anvilkit_render::renderer::standard_material::StandardMaterial;

commands.spawn((
    mesh_handle,
    StandardMaterial::new()
        .with_base_color([1.0, 0.2, 0.1, 1.0])
        .with_metallic(0.0)
        .with_roughness(0.7),
    Transform::default(),
));
```

`render_extract_system` automatically generates `DrawCommand`s from `(MeshHandle, StandardMaterial, GlobalTransform)` queries.

## Asset Server

Supports both synchronous and asynchronous loading:

```rust
use anvilkit_assets::asset_server::AssetServer;

let mut server = AssetServer::new("assets/");

// Synchronous: register intent (no I/O)
let handle = server.load::<Mesh>("models/tree.glb");

// Asynchronous: load file in background (fixed-size thread pool, 1-4 workers)
let async_handle = server.load_async::<Vec<u8>>("textures/atlas.png");

// In your game loop: poll for completed loads
server.process_completed();
for result in server.drain_completed() {
    match result.data {
        Ok(bytes) => { /* Parse and use the loaded data */ },
        Err(err) => { /* Handle error */ },
    }
}
```

## DependencyGraph

`DependencyGraph` tracks parent-child relationships between assets using type-safe `AssetId` handles (not bare `u64`). It is integrated into the `AssetServer` and drives cascade unloading.

| Method | Signature | Description |
|--------|-----------|-------------|
| `add_dependency` | `(&mut self, parent: AssetId, child: AssetId)` | Record that `parent` depends on `child`. |
| `remove_asset` | `(&mut self, id: AssetId) -> Vec<AssetId>` | Remove an asset and return orphaned children (children with no remaining parents). |
| `dependents` | `(&self, id: AssetId) -> &[AssetId]` | List all assets that depend on the given asset. |
| `dependencies` | `(&self, id: AssetId) -> &[AssetId]` | List all assets that the given asset depends on. |

```rust
use anvilkit_assets::dependency::DependencyGraph;

let mut graph = DependencyGraph::new();
graph.add_dependency(scene_id, mesh_id);
graph.add_dependency(scene_id, texture_id);

// Query
let deps = graph.dependencies(scene_id); // [mesh_id, texture_id]
let parents = graph.dependents(mesh_id);  // [scene_id]
```

## AssetCache (Content-Hash Caching)

The `AssetServer` includes a content hash-based cache that deduplicates loads:

- When an asset finishes loading in `process_completed()`, its content hash is computed and stored in the cache.
- Subsequent calls to `load_async` skip the disk read entirely on a cache hit and return the cached data immediately.
- The cache is automatically cleaned up when assets are unloaded.

This is transparent to game code — no API changes are needed to benefit from caching.

## Cascade Unload

`AssetServer::unload(id)` performs a full dependency-aware teardown:

1. Calls `remove_and_cascade` on the `DependencyGraph` to find orphaned children.
2. Cleans up the asset cache, parsed storage, and state tracking for the unloaded asset.
3. Recursively unloads children that have no other parents.

```rust
// Unload a scene — its exclusive meshes and textures are cleaned up automatically
server.unload(scene_id);
```

## Typed Parsed Storage

The `AssetServer` provides typed storage for parsed asset values, bridging raw byte loading with application-level types:

| Method | Signature | Description |
|--------|-----------|-------------|
| `insert_parsed` | `<T: 'static>(&mut self, id: AssetId, value: T)` | Store a parsed asset value of any type. |
| `get_parsed` | `<T: 'static>(&self, id: AssetId) -> Option<&T>` | Retrieve a previously stored parsed value. |

```rust
// After loading raw bytes, parse and store the result
let mesh: Mesh = parse_mesh(&raw_bytes);
server.insert_parsed::<Mesh>(mesh_id, mesh);

// Later, retrieve the parsed mesh
if let Some(mesh) = server.get_parsed::<Mesh>(mesh_id) {
    // Use the mesh
}
```

## Hot Reload

File watching for automatic asset refresh during development (requires `hot-reload` feature). Changes are debounced (200ms) to coalesce rapid file-system events. File deletions (`Remove`) are also reported.

```rust
use anvilkit_assets::hot_reload::FileWatcher;

let mut watcher = FileWatcher::new("assets").unwrap();

// In your game loop (poll_changes takes &mut self):
for path in watcher.poll_changes() {
    if FileWatcher::is_shader(&path) {
        // Rebuild render pipeline with new shader
    } else if FileWatcher::is_texture(&path) {
        // Re-upload texture to GPU
    }
}
```
