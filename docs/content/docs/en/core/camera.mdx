`anvilkit-camera` provides a production-grade camera system with composable ECS components.

## Module Structure

```
anvilkit_camera
├── controller        CameraMode + CameraController
├── input_curve       Dead zone + response curve
├── systems           Core ECS systems (input, mode, effects)
├── plugin            CameraPlugin registration
├── orbit/            Orbit subsystem
│   ├── OrbitState    Distance, target, limits
│   ├── rig           Entity follow + offset + damping
│   └── spring_arm    Collision avoidance (Ray-AABB)
├── effects/          Visual effects subsystem
│   ├── CameraEffects Trauma shake, head bob, FOV
│   ├── noise         Perlin gradient noise
│   └── transition    Smooth camera blending (5 easing types)
└── constraints/      Camera constraints
    ├── look_at       Soft look-at with dead zone
    └── rail          Dolly/path camera (Linear + Catmull-Rom)
```

## CameraController

```rust
use anvilkit_camera::prelude::*;

let controller = CameraController {
    mode: CameraMode::FirstPerson,
    move_speed: 10.0,
    mouse_sensitivity: 0.003,
    base_fov: 70.0,
    input_curve: InputCurve::quadratic(0.05), // 5% dead zone, quadratic response
    ..Default::default()
};
```

The system pipeline reads `InputState::mouse_delta()` automatically — no manual mouse management needed.

## Camera Modes

| Mode | Description | Required Components |
|------|-------------|-------------------|
| `FirstPerson` | Direct yaw/pitch control | — |
| `ThirdPerson` | Orbit around followed entity | `OrbitState` + `CameraRig` |
| `Orbit` | Orbit around fixed point | `OrbitState` |
| `Free` | 6DOF editor/debug camera | — |
| `Rail` | Follow predefined path | `CameraRail` |

## System Pipeline

All systems run in `PostUpdate`, in order:

```
rig → input → rail → mode → spring_arm → look_at → effects → transition
```

## Orbit + Entity Follow

```rust
use anvilkit_camera::prelude::*;

// Spawn camera that follows a player entity
world.spawn((
    CameraController { mode: CameraMode::ThirdPerson, ..default() },
    OrbitState::new(Vec3::ZERO, 8.0)
        .with_distance_limits(3.0, 20.0)
        .with_target_offset(Vec3::new(0.0, 1.6, 0.0)),  // Eye height
    CameraRig::new(player_entity)
        .with_offset(Vec3::new(0.0, 1.6, 0.0))
        .with_damping(10.0),
    SpringArm::new(0.3).with_margin(0.2),  // Collision avoidance
    CameraComponent::default(),
    Transform::default(),
));
```

## Camera Effects (Trauma System)

Uses the Squirrel Eiserloh GDC 2016 trauma system with Perlin noise:

```rust
use anvilkit_camera::effects::CameraEffects;

let mut fx = CameraEffects::default();
fx.head_bob_enabled = true;

// Trigger shake (e.g., on landing, explosion)
fx.add_trauma(0.5);    // Adds to trauma [0,1], decays over time
                         // Shake intensity = trauma^power * noise(time)

fx.fov_target = 10.0;   // Sprint FOV offset (smoothly interpolated)
```

`add_shake()` is kept as a backward-compatible alias for `add_trauma()`.

## Camera Transitions

Smooth blending when switching modes:

```rust
use anvilkit_camera::prelude::*;

// Attach to camera entity
let transition = CameraTransition::new(0.5, EasingType::SmoothStep);

// Before switching mode, capture current state
transition.start(current_pos, current_rot, current_fov);
ctrl.mode = CameraMode::ThirdPerson;
// System automatically blends over 0.5 seconds
```

Available easing types: `Linear`, `SmoothStep`, `SmootherStep`, `EaseInOutCubic`, `EaseOutQuart`.

## Rail Camera

Path-following dolly camera with Catmull-Rom spline:

```rust
use anvilkit_camera::prelude::*;

let rail = CameraRail::new(vec![
    Vec3::new(0.0, 5.0, -10.0),
    Vec3::new(10.0, 3.0, 0.0),
    Vec3::new(0.0, 5.0, 10.0),
    Vec3::new(-10.0, 3.0, 0.0),
])
.with_speed(0.1)
.with_looping(true)
.with_interpolation(RailInterpolation::CatmullRom { tension: 0.5 });
```

## Input Curves

Dead zone and response curve for precise camera control:

```rust
use anvilkit_camera::input_curve::InputCurve;

InputCurve::linear()           // 1:1 response (default)
InputCurve::quadratic(0.05)    // Dampened small inputs, 5% dead zone
InputCurve::cubic(0.1)         // Strong dampening, 10% dead zone
InputCurve::new(0.08, 1.5)     // Custom: 8% dead zone, power 1.5
```

## Gimbal Lock Protection

Third-person and look-at systems automatically handle gimbal lock — when the camera approaches directly above or below the target, an alternative up vector prevents flipping.
